# WhatsApp Sender — UAE bulk messaging

## Архитектура

```
main.py               — точка входа, оркестрация, антиспам-паузы
├── contact_manager.py — чтение CSV/TXT, валидация номеров ОАЭ
├── whatsapp_client.py — Meta Cloud API: отправка с кнопкой-ссылкой
├── message_generator.py — Claude API: вариации текста
├── result_logger.py  — запись результатов в CSV + JSON
└── config.py         — все константы из .env
```

---

## Стек и выбор API

### WhatsApp: Meta Business Cloud API (официальный)

**Плюсы:**
- Официальный — нет риска блокировки аккаунта
- Поддерживает интерактивные сообщения с кнопками-ссылками (CTA URL)
- Надёжный, не зависит от изменений UI WhatsApp Web
- Масштабируется до миллионов сообщений в день

**Минусы:**
- Требует верификации бизнеса в Meta (1–3 дня)
- Маркетинговые шаблоны нужно согласовывать с Meta
- Первые 1000 service conversations в месяц бесплатны, далее платно

**Альтернативы (неофициальные):**
| Инструмент | Плюсы | Минусы |
|---|---|---|
| `whatsapp-web.js` (Node.js) | Бесплатно, все фичи | Риск бана, нужен телефон онлайн |
| Selenium (текущий подход) | Просто | Медленно, XPath ломается |
| Twilio / 360Dialog | Быстрый старт | Платно, посредник |

## Установка и запуск

### 1. Требования

- Python 3.11+
- Аккаунт Meta for Developers с верифицированным бизнесом
- API-ключ Anthropic (claude.ai)

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка окружения

```bash
cp .env.example .env
# Открой .env и заполни токены
```

**Где получить WA_ACCESS_TOKEN и WA_PHONE_NUMBER_ID:**
1. Зайти на [developers.facebook.com](https://developers.facebook.com)
2. Создать приложение → добавить продукт "WhatsApp"
3. WhatsApp → API Setup → скопировать `Phone number ID`
4. Создать системного пользователя в Business Manager → выдать токен

### 4. Подготовка контактов

Файл `contacts.csv` — два столбца:

```csv
phone,name
+971501234567,Алексей
00971521234567,Мария
0551234567,
```

Поддерживаемые форматы номеров ОАЭ:
- `+971 50 123 4567`
- `00971501234567`
- `0501234567`
- `501234567`

### 5. Запуск

```bash
# Тестовый прогон (без реальных запросов)
python main.py --dry-run

# Рассылка без вариаций (один текст для всех)
python main.py --no-variations

# Полноценная рассылка с вариациями через Claude
python main.py

# Другой файл контактов
python main.py --contacts my_numbers.csv
```

## Кнопка с ссылкой в сообщении

Используется тип сообщения `interactive/cta_url`:

```json
{
  "type": "interactive",
  "interactive": {
    "type": "cta_url",
    "body": { "text": "Текст сообщения..." },
    "action": {
      "name": "cta_url",
      "parameters": {
        "display_text": "Присоединиться",
        "url": "https://example.com/event"
      }
    }
  }
}
```

Получатель видит сообщение + синюю кнопку. По нажатию открывается браузер с указанным URL.

> **Важно:** `interactive/cta_url` работает без шаблонов только в рамках 24-часового окна после входящего сообщения от пользователя. Для холодной рассылки нужен согласованный template с `CALL_TO_ACTION` кнопкой.

## Антиспам-логика

| Параметр | Значение по умолчанию | Файл |
|---|---|---|
| Задержка между отправками | 8–20 сек (случайная) | `config.py` |

| Лимит в час | 40 сообщений | `config.py` |

Все константы в `config.py` — можно менять без правки логики.

## Результаты

После запуска в папке `results/` появятся два файла:

**CSV** (открывается в Excel):
```
timestamp,phone,status,message_id,message_text,error
2025-04-01T14:23:01,971501234567,success,wamid.xxx,Привет!...
2025-04-01T14:23:21,971521111111,invalid_phone,,,...
```

**JSON** (полная история для программного анализа):
```json
[
  {
    "timestamp": "2025-04-01T14:23:01",
    "phone": "971501234567",
    "status": "success",
    ...
  }
]
```

Статусы: `success` | `api_error` | `invalid_phone` | `failed`
